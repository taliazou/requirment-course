EasyReminder

Introduction


Our product is an app designed for adults aged 65+.
The application is a system for managing a diary and reminders adapted to this population,
the application will make it possible to schedule events, clinic appointments and more in a simple and easy way.
In addition, the system will enable the provision of smart reminders according to the type of event and the user's preferences.
The purpose of the system is to make it as easy as possible for the user to manage meetings and personal events
and to prevent situations of forgetting and missing these events.

Stakeholders

Main user- elderly population (65+).
Third party entities-Pharmacies, Sports clubs, Community Centers, clinics and more.
Software Developers-system development and maintenance, updates, upgrades and additions.

Questionnaire

The purpose of the questionnaire is to collect detailed information from the target audience of the application that we would like to develop.
The questionnaire included questions about the intended goals of the application and through it we understood the needs and requirements of our main users.

Requirements


Inclusive-
An easy and convenient interface to operate that will allow easy orientation even for those who have difficulty with technology using a variety of languages,
readable visual icons and shortcuts of actions.

Fairness-
The system will allow a voice reading of events, a large font change in the diary and a special color scheme for the color blind.

Privacy-
The private data of the system will be confidential and will not be shown to third party entities.

Alternatives

The goal is to identify potential solutions that meet the project's objectives and requirements and to select the most appropriate one to proceed with.



Quality and Economic criteria

Quality Criteria

Data Security-
The user will only be able to give access to family members. In addition, we will limit the use so that external entities cannot access and use private information (such as crossing a full name with an ID).
Reliability-
1. Make sure that an event that a user enters into the diary is indeed entered with all the relevant details.
2. Smart reminders do occur depending on the classification of the event.
3. Scanning the event using the barcode enters the event properly.
Up-to-date information-
We will make sure that the organization of an event takes place successfully and its details are updated accordingly. As soon as a collision between events occurs - the system knows how to recognize it and send an alert.
Simplicity of system operation-
The application is intended for the adult population, therefore we will emphasize the simplicity of operation which will be expressed by changing the font size as needed, colors and voice readings of events.

Economic Criteria

Development and operating costs-
Server maintenance, cloud usage costs, encryption costs and information security.
